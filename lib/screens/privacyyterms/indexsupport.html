🚀 Comenzando con FlutterPutter

Instalación

FlutterPutter está disponible para múltiples plataformas:

  • Android: Descarga desde Google Play Store
  • iOS: Descarga desde Apple App Store
  • Web: Accede directamente desde tu navegador

Primer Acceso

Al abrir FlutterPutter por primera vez:

  1. Acepta los términos de servicio y política de privacidad
  2. Configura un PIN de seguridad (4-15 caracteres)
  3. Activa la autenticación biométrica si está disponible
  4. Genera tu nickname único para identificación

Sistema de Invitaciones

FlutterPutter utiliza un sistema de invitaciones restringido:

  • Solicita un código de invitación a un usuario existente
  • El código tiene 8 caracteres alfanuméricos
  • Los códigos expiran por seguridad
  • Una vez dentro, puedes invitar a otros usuarios

🔐 Resumen de Seguridad

Cifrado Post-Cuántico

FlutterPutter implementa algoritmos resistentes a computadoras cuánticas:

  • Kyber-768 (ML-KEM): Encapsulación de claves post-cuántica
  • XSalsa20-ChaCha20: Cifrado de flujo de grado militar
  • Curve25519: Intercambio de claves Diffie-Hellman
  • HKDF-SHA256: Derivación segura de claves

Arquitectura Zero-Knowledge

Nuestros servidores nunca ven contenido descifrado:

  • Solo procesan blobs cifrados opacos
  • No tienen acceso a claves de descifrado
  • Imposible acceder a conversaciones incluso con acceso físico
  • Perfect Forward Secrecy garantizado

Protección de Contenido

  • Bloqueo de Capturas: Pantalla negra automática
  • Detección de Screenshots: Notificaciones en tiempo real
  • Sesión Única: Solo un dispositivo activo por usuario
  • Autodestrucción: Mensajes con tiempo de vida configurable

⭐ Guía de Características

Chats Efímeros

Los mensajes se autodestruyen automáticamente:

  • 5 minutos: Para conversaciones ultra-sensibles
  • 30 minutos: Conversaciones cortas
  • 1 hora: Sesiones de trabajo
  • 6 horas: Proyectos de día completo
  • Sin límite: Solo destrucción manual

Videollamadas Seguras

Comunicación P2P con triple cifrado:

  • TLS 1.3 + SSL para transporte
  • Kyber para protección post-cuántica
  • XSalsa20-ChaCha20 para contenido
  • WebRTC con ICE candidates seguros

Verificación de Identidad

Códigos de verificación compartidos:

  • Generación aleatoria criptográficamente segura
  • Compartir por canales externos (WhatsApp, Telegram)
  • Verificación cruzada entre participantes
  • Prevención de ataques man-in-the-middle

🔧 Solución de Problemas

Problemas de Conexión

Si no puedes conectarte:

  • Verifica tu conexión a internet
  • Reinicia la aplicación
  • Comprueba que no hay firewall bloqueando
  • Intenta cambiar de red WiFi a datos móviles

Videollamadas con Problemas

Para mejorar la calidad de video:

  • Asegúrate de tener buena señal
  • Cierra otras aplicaciones que usen cámara
  • Reinicia el dispositivo si persiste
  • Verifica permisos de cámara y micrófono

La App se Cierra Inesperadamente

Si la aplicación falla:

  • Actualiza a la última versión
  • Reinicia el dispositivo
  • Libera espacio de almacenamiento
  • Contacta soporte con detalles del error

🚀 Getting Started with FlutterPutter

Installation

FlutterPutter is available for multiple platforms:

  • Android: Download from Google Play Store
  • iOS: Download from Apple App Store
  • Web: Access directly from your browser

First Access

When opening FlutterPutter for the first time:

  1. Accept terms of service and privacy policy
  2. Set up a security PIN (4-15 characters)
  3. Enable biometric authentication if available
  4. Generate your unique nickname for identification

Invitation System

FlutterPutter uses a restricted invitation system:

  • Request an invitation code from an existing user
  • Code has 8 alphanumeric characters
  • Codes expire for security
  • Once inside, you can invite other users

🔐 Security Overview

Post-Quantum Encryption

FlutterPutter implements quantum-resistant algorithms:

  • Kyber-768 (ML-KEM): Post-quantum key encapsulation
  • XSalsa20-ChaCha20: Military-grade stream cipher
  • Curve25519: Diffie-Hellman key exchange
  • HKDF-SHA256: Secure key derivation

Zero-Knowledge Architecture

Our servers never see decrypted content:

  • Only process opaque encrypted blobs
  • Have no access to decryption keys
  • Impossible to access conversations even with physical access
  • Perfect Forward Secrecy guaranteed

Content Protection

  • Screenshot Blocking: Automatic black screen
  • Screenshot Detection: Real-time notifications
  • Single Session: Only one active device per user
  • Auto-destruction: Configurable message lifetime

⭐ Features Guide

Ephemeral Chats

Messages auto-destruct automatically:

  • 5 minutes: For ultra-sensitive conversations
  • 30 minutes: Short conversations
  • 1 hour: Work sessions
  • 6 hours: Full-day projects
  • No limit: Manual destruction only

Secure Video Calls

P2P communication with triple encryption:

  • TLS 1.3 + SSL for transport
  • Kyber for post-quantum protection
  • XSalsa20-ChaCha20 for content
  • WebRTC with secure ICE candidates

Identity Verification

Shared verification codes:

  • Cryptographically secure random generation
  • Share through external channels (WhatsApp, Telegram)
  • Cross-verification between participants
  • Prevention of man-in-the-middle attacks

🔧 Troubleshooting

Connection Issues

If you can't connect:

  • Verify your internet connection
  • Restart the application
  • Check that no firewall is blocking
  • Try switching from WiFi to mobile data

Video Call Problems

To improve video quality:

  • Ensure good signal strength
  • Close other camera-using applications
  • Restart device if issues persist
  • Verify camera and microphone permissions

App Crashes Unexpectedly

If the application fails:

  • Update to the latest version
  • Restart the device
  • Free up storage space
  • Contact support with error details
